1. Overview

The daemon listens to room audio — through a USB microphone or a line-in / audio-capture input — and every few seconds records a short segment, identifies the music playing, and logs it to a MySQL database. It drives an optional small framebuffer display (album art via feh) and serves this web console for browsing history, statistics, and playlists. Identification uses Shazam, backed by a local fingerprint cache so repeats are recognized without a network call.

Mic or line-in: capture is just an ALSA device. Point AR_ALSA_DEVICE at your input (use List capture devices on the Config page, or arecord -l, to find it — e.g. hw:1,0). Line-in is usually stereo, so set AR_CAPTURE_CHANNELS=2; it's downmixed to mono for recognition. A cleaner line-in signal may also let you lower AR_SILENCE_THRESHOLD_DB.

Everything is configured through AR_* environment variables (see §10); secrets never live in the source. Feature areas that need credentials — Plex, Last.fm scrobbling, Home Assistant, alerts — are inert until configured, so a bare install still records, displays, and serves the archive.

2. How recognition works

The capture loop runs continuously:

  1. Capture & normalize. A RECORD_DURATION-second segment is recorded from ALSA_DEVICE and loudness-normalized. Two buffers ping-pong, so the next segment records while the current one is being identified — there's no dead gap.
  2. Identify. The segment is fingerprinted and checked against the local cache first; on a miss it goes to Shazam.
  3. Vote & lock. Results feed an exponential-moving-average vote keyed on a normalized identity (accents folded, (Remaster)/[Live] and feat. variants collapsed), so spelling variants don't split the vote. A track "locks" once its weight crosses EMA_THRESHOLD.
  4. Enrich. On lock, album/genre/duration that Shazam left blank are filled from Last.fm (with a MusicBrainz duration fallback). Duration is what makes the Now Playing progress dial work. Local-cache hits already carry this and skip the lookup.
  5. Record & show. The play is written to the database (timestamped in UTC), pushed to Now Playing, optionally scrobbled/published, and the art is shown on the display.

When the room goes quiet for SILENCE_RESET_SEGMENTS consecutive segments, Now Playing clears and the vote resets — so the same song heard again after a real gap is logged as a new play, and the listened_seconds of the play just ended is finalized.

3. Local recognition cache no network

Every segment Shazam identifies is fingerprinted with Chromaprint (the fpcalc tool behind AcoustID/MusicBrainz) and the fingerprint is stored against that track. On later segments the app fingerprints the audio and matches it against what it has stored; a close match returns the cached identity and skips Shazam entirely.

Matching stays fast as the library grows via an inverted index — a query only bit-compares against a handful of candidate fingerprints, not the whole corpus. Up to FP_MAX_PER_TRACK reference fingerprints are kept per track (different parts of a song fingerprint differently, so several references raise the hit rate over time). The acceptance bar is FP_MATCH_THRESHOLD (similarity 0–1); same audio scores ~0.9+, unrelated audio ~0.5, so the default is conservative and uncertain segments fall back to Shazam rather than risk a mislabel.

Requires the fpcalc binary (sudo apt install libchromaprint-tools). If it isn't installed, local recognition stays off automatically and every segment goes to Shazam, exactly as before. It's a cache, not a replacement: a brand-new track, or a part of a song not yet fingerprinted, still needs Shazam once.

4. Cover art storage no network

Shazam's cover URLs expire, which would eventually blank out a large archive. When a track is identified, its art is re-encoded to a capped JPEG and stored in two places: a disk cache and the database (cover_blobs, keyed by a hash of the source URL so plays sharing art share one blob). The database copy is the durable source of truth; the disk cache is a fast front layer that is rebuilt from the database if lost.

The console serves art from /cover/<id> and the physical display pulls from the same cache — both resolve disk → database → one-time download. After art has been seen once, neither ever touches the internet for it again. Toggle DB storage with AR_COVER_DB.

5. Web console

Now Playing

Live panel showing art, title, artist, album/genre, and a context line — play #N · last heard <date>. The dial fills against the track duration; when duration is unknown it sweeps in a "scanning" state rather than sitting at zero. The signal lamp lights when audio is currently present.

Tracks

The full archive, one row per play, with infinite scroll. Search by title/artist/album, filter by genre, and sort by recently played, most played, artist, title, album, or first heard. A When row (Today / Yesterday / Last 7 days / custom date range) narrows to a time window — day boundaries are computed in your local timezone against the UTC-stored plays. Merge variants folds remasters/live/feat. versions of the same song into one row. Each row shows whether the track is in your library, streaming search links (Spotify, Apple Music, TIDAL), and a Fix button (see §6). Click a row with an album to reveal album trivia.

History

Chronological play-by-play, grouped by day. Honors the same search/genre/date filters.

Want-list

Distinct tracks you've heard that are not in your library — Plex, or a local music folder if you don't run Plex (see §7) — an acquisition list with streaming links and a Copy list button that copies Artist - Title lines to the clipboard.

Stats

Totals (plays, tracks, artists, albums, hours listened) plus plays/day (average across the days you actually listened), match rate (how much of the music was identifiable — only sustained stretches of truly unrecognizable audio count against it, not brief gaps between or within recognized songs), current day streak, and listening sessions. A 24-hour clock shows when you listen (weighted by minutes, not play counts), followed by top artists/albums/tracks, genres, artists first heard this week, and a GitHub-style calendar heatmap of the last year colored by listening minutes per day.

Selection actions & playlists

Select rows (or "select all matching", which respects the current filters — so "everything played today" is one click) and turn them into a playlist:

  • Add to Plex playlist — creates/append a real Plex audio playlist (Plex only).
  • Create Spotify playlist / Create Tidal playlist — native playlists on those services once connected in Config; tracks matched by search.
  • Download playlist — an M3U of the matches found in your library: Plex stream URLs, or the real file paths when using a local folder.
  • Copy list — plain Artist - Title text for any selection, no library needed, for pasting into a transfer service.

6. Corrections self-healing

Shazam mishears sometimes. The Fix button on any row relabels that track as Artist - Title. This does three things: renames the play, back-relabels every existing play that matched the wrong identity, and stores a raw→canonical mapping that is applied automatically to all future recognitions — so the same misidentification is corrected before it's ever logged wrong again.

The Release button fixes the album a track is attributed to. Recognizers often credit a song to a "Greatest Hits" or compilation; click Release to search Last.fm (falling back to MusicBrainz) for the actual releases it appeared on (originals listed first, compilations/live pushed down), pick the right one, and it's pinned — updating every existing play and all future ones, and steering playlist matching (Plex/Spotify/Tidal prefer the version from that release). If MusicBrainz has nothing, you can type an album name manually.

7. Integrations

All optional; each is inert until its variables are set.

Plex

With AR_PLEX_BASE_URL and AR_PLEX_TOKEN, the console checks each track against your library, streams in-library tracks through a token-free /stream/<id> proxy, exports M3U playlists, and creates/append Plex audio playlists. Large libraries are handled by looking up only your recognized tracks, concurrently (AR_PLEX_CONCURRENCY).

Local music library (no Plex needed)

No Plex? Point AR_LOCAL_LIBRARY_PATH at a folder of music files. It's scanned once (tags via mutagen, filename fallback) into the same title→artist index, so the want-list and the "in library" badge work, and Download playlist emits an M3U of the real file paths — portable and playable in any player. Plex takes precedence when both are set.

Spotify playlists

Create a free app at developer.spotify.com, set AR_SPOTIFY_CLIENT_ID, AR_SPOTIFY_CLIENT_SECRET, and AR_SPOTIFY_REDIRECT_URI (pointing at this console), then click Connect Spotify on the Config page once. Selected tracks are matched by search and added to a new playlist.

Tidal playlists

Enable with AR_TIDAL_ENABLED=1 and click Connect Tidal on the Config page (device login — approve a link.tidal.com URL). No developer app; the session refreshes itself, so you authorize once.

Auto-playlist

Have every newly heard track appended to a running playlist automatically. Set one playlist name and toggle it per service (Spotify, Tidal, Plex) on the Config page. Each distinct track is added once per service, and that dedupe survives restarts, so the playlist becomes a living record of everything the room has played. A track that isn't on a given service is simply skipped there. A track already in the playlist (added by hand or on a previous run) is detected and skipped rather than duplicated.

Adds are queued and retried: hearing a track just enqueues it, and a background worker (every AR_AUTO_PLAYLIST_RETRY_SEC, default 120s) does the actual adding, retrying up to AR_AUTO_PLAYLIST_MAX_ATTEMPTS times. So a service being briefly disconnected, rate limited, or down no longer drops tracks — they wait in the queue and go in once it recovers. The Config page shows the queue depth and whether each service is connected. Sync all heard there queues every track already in your archive, to backfill a playlist after enabling it (or after an outage). For Plex, a track must exist in your Plex library to be added.

Last.fm scrobbling

Set AR_SCROBBLE=1 plus AR_LASTFM_API_KEY, AR_LASTFM_SECRET, and a one-time session key. Get the session key with:

AR_LASTFM_API_KEY=... AR_LASTFM_SECRET=... \
    python -m audio_recognition.scrobble

Follow the printed URL, authorize, and set the printed AR_LASTFM_SESSION_KEY. A play scrobbles once it has run for at least AR_SCROBBLE_MIN_SECONDS and past Last.fm's half-track / 4-minute rule.

Home Assistant / now-playing publish

AR_NOWPLAYING_WEBHOOK receives the current track as JSON on every change (point it at an HA webhook trigger). If paho-mqtt is installed and AR_MQTT_HOST is set, a retained message is also published to AR_MQTT_TOPIC.

Watchdog alerts

Set AR_WATCHDOG_SILENCE_MIN > 0 to fire a one-shot alert when no audio is seen for that many minutes during AR_ACTIVE_HOURS — catching an unplugged stereo or dead capture device. Alerts go to AR_NOTIFY_URL (a JSON webhook, or an ntfy topic URL).

Prometheus metrics

/metrics exposes ar_plays_total, ar_tracks_total, ar_recognition_rate, ar_signal_age_seconds, ar_now_playing, and ar_up for Grafana/alerting.

8. Durability

If the database is unavailable when a play is recorded, the play is appended to a local JSONL spool (AR_DB_SPOOL_FILE) and replayed, in order and with its original timestamp, on the next successful write — so a MySQL restart mid-session doesn't punch holes in the history. Schema migrations run automatically at startup; new columns and tables are created if missing.

10. Configuration

Settings live in a single app-owned file, musicguru.conf, in the app's own directory (next to the audio_recognition package) — this application's file and nothing else's. It's loaded at startup and is authoritative (it wins over the ambient environment). The whole thing is portable: copy the folder anywhere and its musicguru.conf travels with it. A legacy .env in that directory is migrated automatically on first run. The file is git-ignored; a musicguru.conf.example ships in the repo.

Edit from the console: the Config page edits musicguru.conf directly — a grouped form with secrets write-only (masked; blank keeps the current value). Most changes take effect on the next restart; the login username/password apply immediately.

Login is mandatory. On first run the console sends you to a one-time setup page to create a username and password (stored hashed). After that every visit requires that login; a machine token (AR_WEB_TOKEN) still works for API/health callers. You can change the credentials any time from the Config page.

Album art display. Drawing cover art to an attached framebuffer screen is off by default (most installs are headless). Turn it on with the Show album art on the display toggle on the Config page — it takes effect immediately, and Test display paints a test card so you can confirm the screen works without waiting for the next track. The viewer is auto-detected: feh (needs X), then fbi/fbv (draw straight to the framebuffer on a headless Pi). Override with AR_DISPLAY_CMD; failures are logged with the reason.

No restarts. Every setting on the Config page applies the moment you save: the capture device and format are read per segment, service clients reconnect, logging handlers are rebuilt, and the database pool is recreated on demand.

Logging. Logs go to /var/log/musicguru.log (override with AR_LOG_FILE), rotated daily with AR_LOG_BACKUP_DAYS days kept (default 14). If that path isn't writable it falls back to the app directory, then the user state dir. Level is AR_LOG_LEVEL (default INFO).

11. Setup & running

Dependencies

  • System: alsa-utils (arecord), ffmpeg (pydub), libchromaprint-tools (fpcalc, for local recognition), feh + a TrueType font (for the display), and a MySQL/MariaDB server.
  • Python: pip install -r requirements.txt (shazamio, mysql-connector-python, requests, Pillow, pydub, Flask; optional paho-mqtt for MQTT).

Database

Create the database and user, grant privileges, and set AR_DB_PASSWORD. The base recognized_songs table plus all feature tables (corrections, segment_counts, known_tracks, fingerprints, cover_blobs) and the listened_seconds column are created/migrated automatically at startup.

Run

python -m audio_recognition        # recognizer + web console
# console: http://127.0.0.1:8000/

For unattended use, run it under systemd with an EnvironmentFile= holding the AR_* variables.

Access & security

The console binds to loopback by default. To expose it on the LAN, put a reverse proxy in front and set AR_WEB_TOKEN — then every request needs ?token=… or an X-Auth-Token header. The token flows through the console's own links automatically.
